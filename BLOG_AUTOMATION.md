# Blog Automation Setup Guide\n\n## Overview\n\nYour portfolio is now configured to automatically fetch and display your latest Medium articles. When you publish a new article on Medium, it will automatically be included on your website.\n\n## How It Works\n\n### 1. **Automatic RSS Feed Fetching**\n\nThe portfolio Rust application fetches your Medium RSS feed at build time:\n- **Feed URL**: `https://medium.com/feed/@dulanjanalakmal`\n- **Display Limit**: Shows the 5 most recent articles\n- **Auto Update**: Every time the site is rebuilt, it fetches the latest articles from Medium\n\n### 2. **Fallback Articles**\n\nIf the RSS feed cannot be fetched (e.g., network issues), hardcoded fallback articles are displayed to ensure the site always works.\n\n## Automation Guide\n\n### Option A: GitHub Actions (Recommended)\n\nA GitHub Actions workflow is configured to automatically:\n1. Check for new Medium articles every 6 hours\n2. Rebuild your portfolio with fresh content\n3. Deploy to AWS S3 + CloudFront\n4. Invalidate CloudFront cache\n\n**Setup Required:**\n\n1. **Add GitHub Secrets** (Settings → Secrets and variables → Actions):\n   - `AWS_ACCESS_KEY_ID`: Your AWS access key\n   - `AWS_SECRET_ACCESS_KEY`: Your AWS secret key\n   - `CLOUDFRONT_DISTRIBUTION_ID`: Your CloudFront distribution ID\n\n2. **Update the workflow file** (`.github/workflows/update-blog.yml`):\n   - Replace `your-portfolio-bucket` with your actual S3 bucket name\n   - Uncomment the AWS deployment steps when ready\n\n### Option B: Local Build + Manual Deploy\n\n```bash\n# Build the portfolio locally\ncd ~/Documents/project/personal/portfolio\ncargo build --release\n\n# Deploy to AWS S3\naws s3 sync ./dist/ s3://your-bucket-name/ --delete\n\n# Invalidate CloudFront cache\naws cloudfront create-invalidation --distribution-id YOUR_DIST_ID --paths \"/*\"\n```\n\n### Option C: Scheduled Cron Job\n\nAdd a cron job to your server to rebuild weekly:\n\n```bash\n# Edit crontab\ncrontab -e\n\n# Add this line (weekly rebuild on Mondays at 3 AM)\n0 3 * * 1 cd ~/Documents/project/personal/portfolio && cargo build --release && aws s3 sync ./dist/ s3://your-bucket/ --delete\n```\n\n## Testing the Setup\n\n### Test RSS Feed Parsing\n\n```bash\n# Try fetching your Medium RSS feed\ncurl -s https://medium.com/feed/@dulanjanalakmal | head -100\n```\n\n### Test Portfolio Build\n\n```bash\ncd ~/Documents/project/personal/portfolio\ncargo build\n\n# Check if dist/index.html includes your latest articles\ngrep -i \"blog\\|article\" dist/index.html\n```\n\n## Updating Articles\n\n**When you publish a new article on Medium:**\n\n1. **Manual update**: Run `cargo build` locally and deploy\n2. **Automatic update**: GitHub Actions will pick it up within 6 hours\n3. **Instant update**: Trigger the workflow manually via GitHub UI\n\n## Customizing the Update Frequency\n\nEdit `.github/workflows/update-blog.yml` to change the schedule:\n\n```yaml\non:\n  schedule:\n    # Every 6 hours (current)\n    - cron: '0 */6 * * *'\n    # Change to every 12 hours: '0 */12 * * *'\n    # Change to daily at 9 AM UTC: '0 9 * * *'\n```\n\n## Cron Job Reference\n\n- `'0 * * * *'` - Every hour\n- `'0 */6 * * *'` - Every 6 hours (default)\n- `'0 */12 * * *'` - Every 12 hours\n- `'0 9 * * *'` - Daily at 9 AM UTC\n- `'0 3 * * 1'` - Weekly (Mondays at 3 AM UTC)\n\n## Dependencies Added\n\nThe following Rust crates were added to support RSS parsing:\n- `rss = \"0.12\"` - RSS feed parsing\n- `reqwest = \"0.11\"` - HTTP client for fetching feeds\n- `tokio = \"1\"` - Async runtime\n\n## Troubleshooting\n\n### Articles not updating?\n\n1. **Check RSS feed**: `curl https://medium.com/feed/@dulanjanalakmal`\n2. **Rebuild locally**: `cargo build && cat dist/index.html | grep \"blog-card\"`\n3. **Check GitHub Actions logs**: Go to your repository's Actions tab\n\n### Build errors?\n\n```bash\n# Clean build\ncargo clean\ncargo build --release\n```\n\n## Support\n\nFor issues with the automation:\n1. Check GitHub Actions workflow logs\n2. Verify AWS credentials and permissions\n3. Ensure your Medium profile URL is correctly set: `https://medium.com/@dulanjanalakmal`\n"
